令牌的作用

令牌(token)是系统的临时秘钥，相当于账号和密码，用来决定是否允许这次请求和判断这次请求是属于哪一个用户的。它允许你在不提供密码或其他凭证的前提下，访问网络和系统资源，这些令牌将持续存在于系统中，除非系统重新启动。令牌最大的特点就是随机性，不可预测，黑客或软件无法猜测出令牌。

假冒令牌可以假冒一个网络中的另一个用户进行各类操作。所以当一个攻击者需要域管理员的操作权限时候，需要通过假冒域管理员的令牌进行攻击。

用户启用一个进程，用户的凭证就会传递给此进程，进程通过凭证来获得用户能够得到的信息。当机器开启了一些程序就会有对应的进程，进程对应启动程序的用户。

令牌的种类：

  • 访问令牌(Access Token)：表示访问控制操作主体的系统对象。
  • 会话令牌(Session Token)：是交互会话中唯一的身份标识。
  • 密保令牌(Security Token)：又叫做认证令牌或硬件令牌，是一种计算机身份校验的物理设备，例如 U 盾。

Windows 的 Access Token 有两种类型：

  • Delegation Token（授权令牌）：也叫主令牌，它支持交互式会话登录(例如本地用户直接登录、远程桌面登录访问)
  • impersonation token（模拟令牌）：它是非交互的会话(例如使用net use访问共享文件夹)。

注：两种 token 只在系统重启后清除具有Delegation token的用户在注销后，该 Token 将变成Impersonation token，依旧有效。

同样也可以这样理解，当前系统中的某个进程或线程能访问到什么样的系统资源,完全取决于你当前进程是拿着谁的令牌。

假设我们通过exp提权或者永恒之蓝等方式得到的权限即为System，而我们利⽤mimikatz和hashdump无法获得administrator⽤户的密码，那我们只能通过令牌窃取进⾏降权，获得administrator⽤户的shell， 从⽽以administrator⽤户的身份启动某些服务(因为某些服务只能通过administrator⽤户启动)。

令牌模拟

为什么要进行令牌模拟？Token是一个用户的身份标识，拿到某用户的Token即可伪装成某用户。

默认情况下，我们列举令牌，只能列举出当前用户和比当前用户权限更低用户的令牌。令牌的数量取决于当前shell的访问级别，如果当前的shell是administrator或者是system，我们就可以看到系统中的所有的令牌。

普通用户或者普通域用户只能列举或窃取自身的令牌；管理员才能列举或窃取所有用户的令牌。

Access Token 的窃取与利用

Access Token 的窃取与利用需要 administrator 管理员权限。也就是说要提权。

窃取 Access Token 的方法： incognito.exe程序 、InvokeTokenManipulat.ps1脚本 、MSF令牌窃取

环境

域控：08server-dc（win server 2008） IP：192.168.31.164

域成员：12server10（win server 2012） IP：192.168.31.166

域账号：administrator(域管理员)；域账号：test (普通域账号)

攻击机：kali IP：192.168.31.128

incognito的使用

利用过程

假设我们已经getshell 12server10的普通域用户权限test。

1、尝试用普通域用户查看域控文件，可以发现是行不通的。

2、通过incognito列举令牌，可以发现仅能列举出自身的令牌。

所以为了拿下域控，我们就需要要利用这个域管进程进行权限提升，获取域管权限。由于目前是普通域用户是无法注入这个域管进程的，所以需要提权，假设这里通过某种方式提权到了system权限。

3、提权到了system权限也是无法查看域控文件的，因此我们需要通过令牌窃取来获取域管理员权限。

4、通过incognito.exe列举出可利用的令牌。

5、通过incognito.exe伪造administrator令牌，可以发现通过域管理员账户就能访问到域控文件了。

MSF令牌窃取

利用过程

利用msf生成一个后面程序，上传到靶机12server10上。

kali开启监听端口。

1、getshell后使用incognito 模块并列举出可使用的令牌。

伪造令牌，可将权限提升至system。

2、除了可以伪造令牌也可以从进程中窃取令牌。

首先使用 ps 命令列出进程查看进程用户，再使用 steal_token pid 窃取令牌就有对应的权限。

窃取进程的令牌后，获得相应进程的用户权限（由administrator提升至system）。

窃取test用户的令牌，可以发现失败了。

这是由于有些进程虽然是某些用户开启的，但实际这个进程是系统进程因此无法窃取令牌。

有两种方法来窃取test用户的令牌。

  1. 通过窃取system的令牌后，再去窃取系统进程即可成功。

  2. 用test用户执行一个普通进程即可窃取令牌。

    runas允许用户在不切换当前登录账户的前提下，以其他账户权限运行程序。
